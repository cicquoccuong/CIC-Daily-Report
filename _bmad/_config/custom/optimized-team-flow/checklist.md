# Checklist — Quy Trình Phối Hợp Team Tối Ưu

## Phân Loại Task
- [ ] Task đã được phân loại đúng track (Quick Fix / Feature / Epic)
- [ ] Anh Cường đã xác nhận track phù hợp

## Quick Fix (Track A)
- [ ] Đọc context files trước khi code
- [ ] Code fix hoàn tất
- [ ] Test/verify đã chạy
- [ ] CHANGELOG đã cập nhật

## Feature (Track B)
- [ ] Brief & phân tích đã chạy (Mary + Winston + Sally + John + Quinn)
- [ ] Anh Cường đã approve trước khi implement
- [ ] Thiết kế kỹ thuật đã có (nếu cần)
- [ ] UX đã được xem xét (nếu có UI)
- [ ] Test plan đã viết SONG SONG với code
- [ ] Code review đã chạy (Mary)
- [ ] Test suite PASS (Quinn/Murat)
- [ ] Documentation đã cập nhật (Paige)
- [ ] Báo cáo kết quả cho Anh Cường (Bob)

## Epic (Track C)
- [ ] PRD hoặc product brief đã tạo (John)
- [ ] Architecture design đã có (Winston)
- [ ] UX design đã có (Sally)
- [ ] Epic đã breakdown thành stories (Bob)
- [ ] Sprint plan đã tạo
- [ ] Mỗi story chạy qua Track B
- [ ] Retrospective sau sprint

## Quy Tắc Vàng (MỌI track)
- [ ] Đọc CLAUDE.md + SYSTEM_STATE.md đầu session
- [ ] Không skip bước nào trong quy trình
- [ ] Không implement trước khi approve
- [ ] Agents chủ động phát biểu — không ai bị bỏ qua
- [ ] Kết quả chảy giữa agents — không relay qua Anh Cường
- [ ] Paige ghi lại thay đổi cuối mỗi feature
